hexo博客的搭建

hex博客的搭建
Publish Date:   2022-01-27

hexo 介绍

Hexo是一款基于Node.js的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在GitHub和Coding上，是搭建博客的首选框架。

工具准备 ：

node-v14.15.3-x64.msi

Git-2.30.0-64-bit.exe

相关变量的配置

在cmd中

  1. 查看 node ，npm 的版本

    node -v npm -v

  2. 下载cnpm 模块

    npm install -g cnpm –registry=https://registry.npm.taobao.org

  3. 查看 cnpm 版本

    cnpm -v

  4. 下载hexo

    cnpm install -g hexo-cli

  5. 查看 hexo 版本

    hexo -v

在git中

  1. 创建一个空的目录 blog

  2. 在blog中以 git bash here 打开 git

  3. 初始化

    hexo init

  4. 在本地4000端口打开

    hexo s

  5. 创建一篇文章 source/_posts

    hexo n “我的第一篇博客文章”

  6. 本地生成

    hexo g

  7. 安装deploy-git ，也就是部署的命令,这样你才能用命令部署到GitHub

    cnpm install –save hexo-deployer-git

  8. 将hexo部署到GitHub,这一步，我们就可以将hexo和GitHub关联起来，也就是将hexo生成的文章部署到GitHub上，打开站点配置文件 _config.yml，翻到 最后，修改为YourgithubName就是你的GitHub账户.

  9. 部署到远端

    以github为仓， 注册一个GitHub账户
    注册完登录后，在GitHub.com中看到一个New repository，新建仓库，创建一个和你用户名相同的仓库，后面加.github.io，只有这 样，将来要部署到GitHub page的时候，才会被识别，也就是xxxx.github.io，其中xxx就是你注册GitHub的用户名，点击create repository

  10. 生成SSH添加到GitHub ，回到你的git bash中

    git config –global user.name “yourname”
    git config –global user.email “youremail”

    这里的yourname输入你的GitHub用户名，youremail输入你GitHub的邮箱。这样GitHub才能知道你是不是对应它的账户。

    可以用以下两条，检查一下你有没有输对

    git config user.name
    git config user.email

    然后创建SSH,一路回车

  11. 生成SSH

    ssh-keygen

    ssh，简单来讲，就是一个秘钥，在c:用户/xxx/.ssh，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

  12. 上传id_rsa.pub的信息

    在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key把你的id_rsa.pub里面的信息复制进去

  13. 在gitbash中,查看是否成功

    ssh -T git@github.com

  14. 将hexo部署到GitHub

    这一步，我们就可以将hexo和GitHub关联起来，也就是将hexo生成的文章部署到GitHub上，打开站点配置文件 _config.yml，翻到 最后，修改为YourgithubName就是你的GitHub账户.

  15. 执行

    hexo clean

    hexo generate

    hexo deploy

选择主题

  1. 以 matery 为例

    git clone https://github.com/blinkfox/hexo-theme-matery.git themes/matery

  2. 在 _config.yml 中的 themes：改为 themes: matery

  3. 在hexo中下载图片插件

    cnpm install hexo-asset-image –save

  4. 在_config.yml配置文件中，修改为 post_asset_folder: true

  5. 新建一篇文章

    hexo n “hahahaha”

  6. 在根目录 blog\source_posts 文件夹下会出现一个hahaha.md 和hahaha文件夹

  7. 将文档中的图片引入到同名的文件夹中即可。

常见问题的问题的解决

  1. git出错：“Please make sure you have the correct access rights and the repository exists.

    ssh 需要重置

    1. 重置用户名和邮箱

      1. git config –global user.name “yourname”

      2. git config –global user.email “your@email.com“

        yourname: 你要设置的邮箱

        your@email: 你要设置的邮箱

    2. 删除 .ssh文件夹下的 known_hosts

    3. git 输入命令

      $ ssh-keygen -t rsa -C “your@email.com“ (‘自己设置的邮箱地址’)

      一路回车或yes

      然后系统会自动在 .ssh文件夹下生成两个文件，id_rsa 和 _rsa.pub,打开将里边的东西全部复制

    4. 打开 https://github.com/,登录你的账户，进入设置

      进入ssh设置

      在 key 中将复制的内容粘贴进去

      点击 add ssh key

    5. 在 git 中输入命令 ： ssh -T git@github.com

      然后会跳出一堆话

      输入 yes 或 回车，即可解决。
